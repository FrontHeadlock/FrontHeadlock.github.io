# Tech / Contact 제거 spec

## 목표

이번 작업이 끝났을 때 포트폴리오 화면에서 `Tech Stack` 섹션과 `Contact` 섹션이 제거되어, 더 짧고 집중도 높은 흐름으로 읽히게 해야 합니다.

구체적으로 달라져야 하는 점:

- 상단 내비게이션에서 `Tech Stack`, `Contact` 탭이 사라질 것
- 본문에서 `Tech Stack` 섹션이 사라질 것
- 본문에서 `Contact` 섹션이 사라질 것
- 남은 섹션 흐름이 어색하지 않게 이어질 것

## 문맥

현재 화면은 전체 톤과 레이아웃 완성도는 충분히 좋지만, 하단의 `Tech Stack`과 `Contact` 두 섹션이 정보 밀도를 높이기보다 흐름을 늘리는 역할에 가깝습니다.

사용자 판단 기준:

- 현재 포트폴리오는 `Hero`, `About`, `Experience`, `Projects` 중심으로도 충분히 강점이 전달됨
- `Tech Stack`, `Contact`는 없어도 전체 메시지가 약해지지 않음
- 오히려 제거했을 때 더 단단하고 채용용 이력서 사이트에 가까워짐

이번 변경은 “시각 리디자인”이 아니라 “정보 구조 압축”에 가깝습니다.

## 제약

이번 차수에서는 다음을 하지 않습니다.

- 색상, 타이포, 배경 효과의 전면 재설계
- Hero / About / Experience / Projects 콘텐츠 재작성
- 프로젝트 상세 구조 변경
- 새로운 섹션 추가
- 연락처 정보를 다른 섹션으로 재배치하는 대규모 개편

즉, 이번 작업은 `Tech Stack`과 `Contact`를 제거하고, 그 제거로 인해 생기는 빈 흐름만 정리하는 최소 변경 작업입니다.

## 완료 조건

아래 조건이 모두 만족되면 완료로 봅니다.

1. 상단 내비게이션에 `Tech Stack`, `Contact` 탭이 더 이상 보이지 않을 것
2. 본문에 `Tech Stack` 섹션이 렌더되지 않을 것
3. 본문에 `Contact` 섹션이 렌더되지 않을 것
4. 남은 섹션 순서는 `Home -> About -> Experience -> Projects`로 정리될 것
5. 섹션 제거 후에도 레이아웃 간격과 스크롤 흐름이 어색하지 않을 것
6. 테스트와 빌드가 다시 통과할 것

## 변경 범위

직접 영향 범위:

- 헤더 내비게이션 항목
- 메인 페이지 섹션 구성
- 해당 섹션에 연결된 테스트

예상 파일:

- `src/components/Header/Header.tsx`
- `src/App.tsx`
- `src/App.test.tsx`

간접 영향 가능 파일:

- `src/components/TechStack/TechStack.tsx`
- `src/components/Contact/Contact.tsx`

단, 이번 차수에서는 위 두 파일을 굳이 삭제하지 않아도 됩니다. 우선은 렌더 경로와 내비게이션에서 제외하는 것이 1차 목표입니다.

## 설계 결정

### 1. 제거 방식

`Tech Stack`, `Contact`는 “숨김 처리”가 아니라 “렌더 경로에서 제외”합니다.

이유:

- DOM에 남겨두면 향후 회귀 가능성이 커짐
- 현재 목적은 UI 축소가 아니라 정보 구조 축소이므로, 실제 화면 흐름에서 빠지는 것이 맞음

### 2. 내비게이션 처리

헤더 항목은 아래 4개만 남깁니다.

- `Home`
- `About`
- `Experience`
- `Projects`

### 3. 레이아웃 처리

`Projects`가 마지막 섹션이 되므로, 하단 여백과 종료감이 자연스럽도록 마지막 구간 spacing은 점검합니다.

### 4. 데이터 처리

이번 차수에서는 `skills`, `profile.links`, `deliveryMeta` 같은 데이터 파일을 즉시 삭제하지 않습니다.

이유:

- 이번 변경의 본질은 “화면 제거”이지 “데이터 모델 정리”가 아님
- 실제 삭제는 후속 정리 작업으로 분리하는 편이 안전함

## 구현 원칙

1. 가장 작은 변경으로 섹션 제거
2. 내비게이션과 본문 렌더를 먼저 맞춤
3. 테스트를 함께 갱신
4. 마지막에 빌드와 레이아웃 흐름 확인
