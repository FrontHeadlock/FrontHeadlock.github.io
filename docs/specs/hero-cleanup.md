# Hero 정리 및 Matrix Rain 보정 spec

## 목표

이번 작업이 끝났을 때 첫 화면이 더 간결해지고, 배경의 `matrix rain`이 한 번만 지나가고 멈춘 것처럼 보이지 않도록 지속적으로 흐르는 상태가 되어야 합니다.

구체적으로 달라져야 하는 점:

- `Resume PDF` 항목이 화면에서 제거될 것
- Hero 상단의 `DevOps Portfolio` 배지가 제거될 것
- Hero 내부의 `> whoami` 터미널 블록이 제거될 것
- `matrix rain`이 초기 몇 초 뒤 사라진 것처럼 보이지 않고 계속 흐를 것

## 문맥

현재 포트폴리오는 전체 구조와 밀도는 충분히 좋지만, Hero 상단의 장식 요소가 정보 전달을 늘리기보다 시선을 분산시키는 부분이 있습니다.

현재 문제는 크게 두 가지입니다.

1. Hero 정보 밀도 문제
- `DevOps Portfolio` 배지와 `whoami` 블록은 터미널 감성은 전달하지만, 실제 채용용 첫 화면에서 이름과 핵심 요약보다 먼저 읽힐 필요는 없음
- `Resume PDF`는 실제 파일이 연결되지 않은 상태였고, 현재 사이트 흐름에서는 제거하는 편이 더 자연스러움

2. 배경 애니메이션 지속성 문제
- 현재 `matrix rain`은 처음 몇 초 동안만 눈에 띄고, 이후에는 사실상 멈추거나 거의 보이지 않는 것처럼 인식됨
- 배경 애니메이션은 본문을 가리지 않으면서도 페이지 전체에서 계속 살아 있어야 함

이번 변경은 새로운 연출을 추가하는 작업이 아니라, 이미 있는 Hero와 배경 효과를 더 정확한 방향으로 정리하는 작업입니다.

## 제약

이번 차수에서는 다음을 하지 않습니다.

- Hero 전체 레이아웃 전면 재설계
- 이름, 소개 문구, 핵심 지표 카드의 콘텐츠 재작성
- 색상 체계와 타이포 시스템 변경
- 새로운 링크 항목 추가
- matrix rain을 과장된 전면 효과로 키우는 변경
- 성능 최적화를 이유로 canvas 효과 자체를 제거하는 방향

즉, 이번 작업은 `불필요한 Hero 요소 제거`와 `배경 애니메이션 지속성 보정`에 한정합니다.

## 완료 조건

아래 조건이 모두 만족되면 완료로 봅니다.

1. Hero 링크 영역에서 `Resume PDF` 버튼이 더 이상 렌더되지 않을 것
2. Hero 상단에서 `DevOps Portfolio` 배지가 더 이상 보이지 않을 것
3. Hero의 `> whoami` 터미널 스타일 블록이 더 이상 보이지 않을 것
4. 이름, 소개 문구, 핵심 지표 카드는 정상적으로 유지될 것
5. `matrix rain`이 페이지 진입 후에도 계속 흐르는 배경 애니메이션으로 보일 것
6. 배경 효과가 본문 가독성을 해치지 않을 것
7. 테스트와 빌드가 다시 통과할 것

## 설계 결정

### 1. Resume PDF 처리

이번 차수에서는 `Resume PDF`를 비활성 버튼으로 남기지 않고, Hero 링크 목록에서 완전히 제거합니다.

이유:

- 실제 파일이 없는 링크를 남겨둘 이유가 없음
- 이미 `Contact` 섹션도 제거된 상태이므로, Hero 링크는 실제 연결 가능한 항목만 남기는 편이 일관적임

### 2. Hero 장식 요소 정리

다음 두 요소를 제거합니다.

- `DevOps Portfolio` 배지
- `> whoami` 블록

Hero는 아래 정보만 바로 보이는 구조로 단순화합니다.

- 이름
- 한 줄 요약
- 실제 링크
- 핵심 지표

### 3. Matrix Rain 동작 원칙

`matrix rain`은 한 번 재생되는 인트로가 아니라, 페이지 전체에서 지속적으로 흐르는 저강도 배경이어야 합니다.

보정 방향:

- 컬럼 위치가 캔버스 하단에 도달해도 다시 위에서 자연스럽게 순환할 것
- 초기 몇 초 뒤 정지한 것처럼 보이지 않도록 프레임 갱신이 계속 유지될 것
- 현재의 낮은 존재감은 유지하되, 시각적으로 “살아 있는 배경”이라는 인식은 유지할 것

### 4. 정보 우선 원칙 유지

배경 애니메이션이 개선되더라도, Hero의 주인공은 여전히 텍스트와 지표입니다.

우선순위:

1. 이름과 역할 인지
2. 구조적 문제 해결 요약
3. 수치 기반 성과
4. 배경의 분위기 보조

## 구현 원칙

1. Hero에서 제거할 요소는 숨김이 아니라 렌더 경로에서 제외
2. 실제 링크만 남기고 placeholder 링크는 제거
3. matrix rain은 canvas 루프와 컬럼 재순환 로직을 최소 수정으로 보정
4. 가독성을 해치지 않는 범위에서만 애니메이션 존재감을 유지
5. 테스트는 제거 대상과 유지 대상 둘 다 확인하도록 갱신
