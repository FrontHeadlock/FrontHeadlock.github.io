# 포트폴리오 가시성 이슈 작업 계약

## 목표

이번 작업이 끝났을 때 웹페이지의 모든 주요 컴포넌트가 초기 진입 후 잠깐 보였다가 사라지는 문제 없이, 기본 값 그대로 지속적으로 보여야 합니다.

이번 차수의 최종 목표는 다음과 같습니다.

- 첫 진입 직후 보이던 `Hero`, `About`, `Experience`, `Projects`, `Tech Stack`, `Contact` 영역이 몇 초 뒤에도 그대로 유지될 것
- 배경 `matrix rain`은 콘텐츠를 덮지 않고, 장식으로만 남을 것
- 현재보다 matrix rain 가시성은 조금 더 올라가되, 본문 가독성을 해치지 않을 것
- Safari를 포함한 실제 브라우저에서 재현되지 않는 상태로 수습할 것

## 문맥

### 현재 문제

현재 `/Users/kyum/Desktop/vibe/cv/resume`의 포트폴리오 웹페이지는 처음 접속하면 정상적으로 내용이 보이지만, 약 3초 정도 지난 뒤 화면의 주요 콘텐츠가 모두 사라지고 거의 배경만 남는 상태로 바뀝니다.

사용자 기준으로는 다음처럼 보입니다.

1. 초기 접속 직후:
   - 네비게이션, 히어로 텍스트, 메트릭 카드, 본문 섹션들이 정상적으로 보임
2. 약 3초 후:
   - 주요 컴포넌트들이 더 이상 보이지 않음
   - 배경의 어두운 matrix 계열 레이어만 남은 것처럼 보임

### 재현 위치

- 로컬 프리뷰 URL: `http://127.0.0.1:4173/`
- 사용자 보고 기준 브라우저: Safari에서도 동일하게 재현
- 사용자 판단 기준: 캐시 문제나 일회성 렌더 오류가 아니라 런타임 로직 문제로 의심됨

### 현재까지 수집된 근거

- `Reveal` 공통 래퍼는 한 번 노출되면 다시 `revealed=false`로 돌아가지 않도록 이미 수정됨
- `matrix canvas`는 `-z-10`과 `isolate` 기반으로 콘텐츠 뒤로 내린 상태
- `test` 및 `build`는 통과하고 있음
- 그럼에도 실제 브라우저에서는 일정 시간 후 화면이 비어 보이는 현상이 남아 있음

이로 인해 단순 타입 오류나 테스트 환경 한정 문제보다, 실제 브라우저 런타임에서 발생하는 상태 변화 또는 레이어링/애니메이션 상호작용 문제가 더 가능성이 높습니다.

## 제약

이번 차수에서는 문제의 원인을 먼저 좁히는 것이 우선이며, 다음은 하지 않습니다.

- 디자인 전면 개편
- 섹션 구조 전체 재작성
- 애니메이션 시스템 전체 제거
- Framer Motion 전체 제거
- 데이터 구조 대규모 변경
- 프로젝트 콘텐츠 재작성

즉, 이번 차수는 “포트폴리오를 더 예쁘게 바꾸는 작업”이 아니라 “보이던 콘텐츠가 사라지는 회귀를 원인 순서대로 분석하고 최소 수정으로 고치는 작업”입니다.

## 완료 조건

이번 이슈는 아래 조건이 모두 만족되면 완료로 봅니다.

1. `http://127.0.0.1:4173/`에서 접속 후 3초 이상 기다려도 콘텐츠가 사라지지 않을 것
2. `Hero`, `About`, `Experience`, `Projects`, `Tech Stack`, `Contact`의 주요 텍스트와 카드가 계속 보일 것
3. 배경 matrix rain은 콘텐츠 위에 올라오지 않고 뒤에서만 보일 것
4. matrix rain opacity 조정 이후에도 본문 가독성이 유지될 것
5. 테스트와 빌드가 다시 통과할 것

## 현재 문제 정의

이 이슈는 “컴포넌트가 렌더되지 않는다”보다 “처음에는 렌더되지만, 이후 어떤 런타임 변화로 인해 결과적으로 안 보이게 된다”에 가깝습니다.

따라서 문제는 다음 중 하나일 가능성이 큽니다.

- 콘텐츠 자체가 DOM에서 제거되거나 `opacity/transform/display` 상태가 바뀌는 경우
- 배경 레이어가 콘텐츠를 시각적으로 덮는 경우
- 특정 훅이 일정 시간 뒤 전체 섹션 또는 상위 레이아웃의 상태를 다시 바꾸는 경우
- 브라우저에서만 발생하는 프레임/뷰포트/애니메이션 계산 상호작용이 있는 경우

## 원인 후보 우선순위

### 후보 1. 상위 레이아웃 또는 공통 모션 래퍼의 런타임 상태 재계산

근거:

- “몇 초 후 사라짐”은 단발성 초기화보다 타이머, 옵저버, 애니메이션 재평가와 더 잘 맞음
- `Reveal`, `AnimatePresence`, `motion.*`가 공통 섹션 래퍼에 걸려 있어 영향 범위가 큼
- 한 번에 거의 전체 콘텐츠가 비어 보인다는 점은 상위 래퍼 문제와 잘 맞음

우선 확인 대상:

- `src/components/shared/Reveal.tsx`
- `src/hooks/useScrollReveal.ts`
- `src/lib/motion.ts`
- Framer Motion이 실제 브라우저에서 섹션 상태를 다시 초기값으로 보내는지 여부

### 후보 2. 배경 matrix canvas와 본문 레이어 충돌

근거:

- 사용자가 본 화면이 “배경만 남는 상태”와 유사함
- canvas는 `requestAnimationFrame`으로 계속 갱신되며, stacking context나 fixed layer 계산 문제가 있으면 실제 브라우저에서만 다르게 보일 수 있음
- Safari는 fixed + opacity + mask + stacking 조합에서 다른 렌더 결과를 낼 여지가 있음

우선 확인 대상:

- `src/components/shared/MatrixRainCanvas.tsx`
- `src/hooks/useMatrixRain.ts`
- `src/styles/matrix-effects.css`
- `App.tsx`의 root stacking context

### 후보 3. 타이핑 / 디코드 훅이 상위 렌더를 반복 유발하면서 시각 상태를 깨뜨리는 경우

근거:

- 화면에 시간 기반 훅이 여러 개 존재함
- `setTimeout`, `setInterval`, `requestAnimationFrame`이 함께 쓰이고 있음
- 특정 상태 변경이 상위 모션 상태와 결합되면 초기값으로 보이는 문제가 발생할 수 있음

우선 확인 대상:

- `src/hooks/useTypingEffect.ts`
- `src/hooks/useDecodeText.ts`
- `src/hooks/useMatrixRain.ts`

### 후보 4. 실제 DOM은 남아 있지만 CSS나 transform 때문에 화면 밖으로 밀리거나 투명해지는 경우

근거:

- 테스트는 통과하지만 실제 브라우저에서 사라지는 현상은 CSS 계산 결과 차이일 가능성도 있음
- `opacity`, `transform`, `mask-image`, `backdrop-blur`, fixed positioning 조합은 브라우저별 편차가 있을 수 있음

우선 확인 대상:

- 실제 DOM 존재 여부
- 섹션 wrapper의 computed style
- opacity / transform / z-index / display / visibility 값

## 분석 원칙

이번 작업은 아래 순서로 진행합니다.

1. “무엇이 사라지는가”를 추측하지 않고 확인한다
2. 실제 브라우저에서 사라진 뒤 DOM이 남아 있는지 먼저 확인한다
3. 남아 있다면 스타일/레이어 문제로 본다
4. 사라진다면 상태/조건부 렌더 문제로 본다
5. 가장 영향 범위가 큰 공통 래퍼부터 차례대로 좁힌다
6. 한 번에 여러 원인을 동시에 바꾸지 않는다

## 개선 접근 방식

### 1단계. 증상 구분

먼저 아래를 분리 확인해야 합니다.

- DOM이 실제로 제거되는지
- DOM은 남아 있는데 opacity 또는 transform만 바뀌는지
- canvas 또는 다른 fixed layer가 위를 덮는지

이 단계가 끝나야 “애니메이션 문제”인지 “레이어 문제”인지 확정할 수 있습니다.

### 2단계. 공통 레이어 차단 실험

가장 영향 범위가 큰 공통 요소를 하나씩 비활성화해 재현 여부를 확인합니다.

우선순위:

1. `MatrixRainCanvas`
2. `Reveal`
3. 타이핑/디코드 훅

목적:

- 어떤 공통 레이어를 끄면 사라짐이 멈추는지 확인
- 한 번에 하나씩만 끄고 재현성을 비교

### 3단계. 원인 확정 후 최소 수정

원인이 확인되면 그 지점만 최소 수정합니다.

예상되는 수정 종류:

- `motion` 상태를 1회성 고정 상태로 단순화
- Safari에서 문제를 만드는 fixed/mask/opacity 조합 제거 또는 축소
- 배경 캔버스 렌더 루프와 레이어 순서 재조정
- 특정 시간 기반 훅의 재실행 조건 제거
